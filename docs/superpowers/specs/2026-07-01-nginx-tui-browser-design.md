# nginx 目录浏览下载 TUI — 设计文档

日期：2026-07-01

## 概述

一个单文件 Python 脚本，接收一个开启了 `autoindex on;` 的 nginx 静态文件服务器 URL，在终端里以全屏 TUI 浏览其目录结构，支持进入子目录、返回上一级、以及回车/鼠标点击下载文件到本地指定目录，下载时显示进度。

## 目标 / 非目标

**目标**
- 浏览 nginx autoindex 生成的目录列表，支持进入子目录、返回上一级
- 回车或鼠标左键点击文件条目触发下载，显示下载进度
- 展示的文件名/大小/修改时间忠实于 nginx 返回的真实数据；列表的排版/配色是脚本自己设计的（不照抄原始 `<pre>` 文本排版）
- 零第三方依赖，单文件即可运行（Python 3 标准库：`curses`/`urllib`/`html.parser`/`argparse`）

**非目标**
- 递归下载整个目录
- 需要登录认证（Basic Auth / Cookie）的服务器
- 自定义 CA 证书的 HTTPS（默认使用系统默认 SSL 上下文；`--insecure`/`-k` 只是跳过证书校验，不做自定义 CA 信任链）
- 排序/过滤/搜索等高级列表操作
- 断点续传

## 架构与组件

- `Entry`（dataclass）：`name`（unquote 后的文件/目录名，目录含尾部 `/`；再经过控制字符净化，见下）、`href`（原始链接，用于拼 URL）、`url`（与当前页面 base URL 拼接后的绝对地址）、`is_dir`（bool）、`size_bytes`（`Optional[int]`，精确字节数，目录或大小非精确数字时为 `None`）、`mtime`（`Optional[str]`，格式化为 `YYYY-MM-DD HH:MM`；解析失败则保留原始字符串兜底，兜底字符串同样经过 `name` 那套控制字符净化——正则日期与时间之间的 `\s+` 能匹配到字面制表符，未净化会让它随兜底字符串一路带进 curses 渲染）、`size_raw`（`Optional[str]`，默认 `None`；仅在服务端给的是 `autoindex_exact_size off` 那种四舍五入单位形式如 `24M`/`1.2K` 时才有值，展示层原样显示这个字符串，不换算成字节数）
- `parse_index(html: str, base_url: str) -> List[Entry]`：
  - 用 `html.parser.HTMLParser` 提取所有 `<a href="...">` 及其位置，保证导航始终可用（不依赖服务端返回文本的具体格式）；出现未闭合就被新 `<a>` 打断的畸形嵌套标签时，会先把还没收尾的外层锚点原样记录下来，再开始追踪新的一层，不会静默丢弃外层
  - 页面中一个 `<a>` 都提取不到时返回空列表（等同于空目录），不报错——不同服务端实现对"空目录"的表示不一致（nginx 空目录仍带 `../` 链接，但例如 Python `http.server` 生成的空目录列表页是零 `<a>` 标签），无法用链接数量可靠区分"真的是空目录"和"根本不是目录列表页"
  - 名称统一取 `unquote(href)`，不用链接文本——因为超长文件名 nginx 会在链接文本里截断显示（`..&gt;`），href 是完整的；unquote 后会把其中的 C0 控制字符（换行、制表符、回车等，服务端一个真实存在的文件名就能带出这些字节）替换成占位符 `�`，否则这些字节会被 curses 当成真实的光标移动指令，把一整行拆成两行、弄乱后续所有行的排版
  - 跳过父目录项（`href` 为 `../`、`/` 或链接文本为 `..`），返回上级由脚本自己的历史栈管理
  - 对每个 `<a>` 标签所在的原始文本行，用正则尽力解析出其后跟着的日期（`%d-%b-%Y %H:%M`）和大小；大小允许纯数字（精确字节数，写入 `size_bytes`）或 `autoindex_exact_size off` 时的 `12.3K`/`4.1M` 这类四舍五入单位形式（写入 `size_raw`，原样展示，不换算成字节数）；日期或大小整体解析不到就把对应字段置空，不影响该条目正常显示和下载；这一步按正则原文抓 `href` 做 key，会先做一次 HTML 实体反转义再去查表匹配锚点提取阶段（`HTMLParser` 已自动实体解码）拿到的 `href`，否则页面把 `&` 写成 `&amp;amp;` 这类被再转义过一次的 href 会查不到 size/mtime
- `fetch_index(url, insecure=False) -> Tuple[str, str]`：`urllib.request` 拉取 HTML，设超时，返回 `(html, final_url)`；`final_url` 是重定向后的最终 URL（nginx 对缺尾部斜杠的目录请求会 301 重定向），调用方必须用它作为 `parse_index` 的 base URL 和历史栈里存的地址，不能用请求时的原始 URL；响应体超过 10MB（正常的目录列表页远小于这个数字）时直接报错放弃，防止一个实际不是目录列表的超大响应把整个内容读进内存卡死界面；HTTP/网络错误交给调用方处理（不在这一层吞异常）；`insecure=True` 时用不校验证书/主机名的 SSL 上下文
- `download_file(url, dest_path, progress_cb, insecure=False)`：流式分块（64KB）写入 `dest_path + ".part"`，成功后 rename 成目标文件名；写完后会校验已接收字节数与 `Content-Length` 是否一致，连接提前断开时 `HTTPResponse.read()` 只会悄悄返回空字节而不抛异常，不做这个校验的话会把一个残缺的文件当成下载成功提交；`Content-Length` 是负数（畸形响应）时当作没有这个头处理，不然完整下载也会被这个校验误判成"不完整"；异常或 `KeyboardInterrupt` 时清理 `.part` 残留

## 导航：历史栈

`BrowserApp` 维护一个栈，每个栈帧是 `(url, entries, selected_index, scroll_offset)`：
- 进入目录：`fetch_index` + `parse_index` 后压入新栈帧
- 返回上级（`Backspace`/`←`/`u`/`Esc`）：弹出当前栈帧，直接用上一帧缓存的 `entries` 重新渲染，**不重新发请求**，并恢复之前的光标位置
- 栈只剩一帧（初始 URL）时再按返回：状态栏提示"已在根目录"，不做操作

## 界面与展示

- 顶部：当前路径面包屑（从当前 URL 派生，对 URL 做 percent-decode 以可读原样显示中文等字符）；路径超出终端宽度时从中间省略（保留开头的协议/主机和结尾最深一级目录名，两者通常都比中间的路径片段更有用）；有颜色支持时用固定的白字蓝底配色（而不是对终端默认色做反色），避免暗色终端主题下对比度不足；无颜色支持时退化为 `A_REVERSE`
- 列表区：固定列布局，自己设计的配色和对齐方式——
  - 名称列（自适应宽度，超长按脚本自己的规则截断，不是照搬 nginx 的截断风格）
  - 大小列（右对齐，`size_bytes` 换算成人类可读单位如 `12.3K`/`4.1M`；没有精确字节数但服务端给了四舍五入单位形式时显示 `size_raw` 原文；两者都没有或目录该列显示 `-`；过长时截断到列宽，不会把行撑爆换行）
  - 修改时间列（右对齐，统一格式 `YYYY-MM-DD HH:MM`；解析失败则原样显示 nginx 给的字符串，同样截断到列宽）
  - 目录与文件用不同颜色/样式区分（有 `curses` 颜色支持时用配色，类似 `ls`；无颜色支持的终端退化为仅用 `A_BOLD`）
  - 选中行整行反色高亮
- 底部：常驻按键提示行，用于帮助提示 / 下载进度条 / 错误信息 / 覆盖确认交互（通过居中高亮的临时提示覆盖显示）；有颜色支持时用固定的青色粗体，不用 `A_DIM`（会降低对比度，暗色终端主题下可能看不清）；无颜色支持时退化为 `A_BOLD`；这类临时提示文字超出终端宽度时也从中间省略，而不是从尾部截断——提示文字里常嵌有文件/目录名，名字很长时尾部截断会把"(y/n)"确认提示、Ctrl-C 取消提示、下载百分比等挤出屏幕，中间省略能保住这些结尾部分；提示文字统一在写入状态栏时做控制字符净化（同 `name` 那套），不止是名字/路径——网络异常的 `str(exc)`（如 HTTP 状态行的 reason phrase）本身也允许带字面制表符等控制字节，不净化会原样冲进 curses 渲染
- 常驻按键提示行本身（"↑/↓/j/k 移动 ... q/Q 退出"）全长 114 列，同样从中间省略而不是从尾部截断——尾部截断会导致几乎所有正常终端宽度下都看不到最后的 "q/Q 退出" 退出提示，中间省略至少能保住开头的移动提示和结尾的退出提示

## 交互

| 按键 | 行为 |
|---|---|
| `↑`/`↓`、`j`/`k` | 移动选择 |
| `PageUp`/`PageDown` | 翻页 |
| `Enter`、鼠标左键点击某行 | 目录→进入；文件→下载 |
| `→` | 仅目录→进入；选中文件时不做任何操作（不触发下载） |
| `Backspace`/`←`/`u`/`Esc` | 返回上级（历史栈弹出） |
| `r`/`R`/`F5` | 刷新当前目录（重新请求并解析，不影响历史栈） |
| `q` | 退出 |

鼠标点击通过 `curses.mousemask(curses.ALL_MOUSE_EVENTS)` 启用，依赖终端本身支持鼠标上报（如 iTerm2、多数现代终端模拟器）；不支持时键盘操作不受影响；终端太小（见错误处理一节）导致列表根本没画出来时，点击会被直接忽略，不会误触发看不见的那一行。

## 下载行为

- 本地文件名 = `unquote(href)` 的 basename 经与显示同一套控制字符净化（C0 控制字节替换为 `�`）之后的结果——不单独区分"显示用"和"落盘用"两条路径，避免服务器文件名里的裸控制字节被原样写进本地文件系统；副作用是两个仅控制字符不同的服务器文件名会净化成同一个本地文件名，此时后一个下载会撞上前一个已存在的文件，走下面的覆盖确认流程（不会静默覆盖）。basename 为空或是 `.`/`..`（畸形/构造的 href）时直接状态栏报错拒绝下载，不进入覆盖确认流程
- 目标路径已经是一个目录：直接状态栏报错拒绝下载，不进入覆盖确认流程——如果走了确认流程答 `y`，会先把整个文件下载完，最后 rename 到目标位置时才失败，白白浪费一次完整下载
- 目标路径或其 `.part` 暂存文件已存在（用 `lexists` 判断，不用 `exists`——`exists` 对失效的软链接会返回 `False`，用它判断会让一个悬空软链接绕过确认直接被覆盖）：状态栏内联提示 `文件已存在，覆盖？(y/n)`，按 `y` 才开始下载，按 `n`/`Esc`/其他任意键取消并返回浏览界面（不报错）；提示文案会区分是目标文件本身存在还是只有 `.part` 暂存文件存在，避免"文件已存在"这句话在目标文件其实还不存在时误导用户；此时 `Esc` 只表示取消，不触发返回上级目录；等待确认期间收到终端 resize 会重新收敛滚动位置并重绘，收到鼠标事件会正常消费掉（避免残留在鼠标事件队列里影响后续点击），两者都不算作按键回答
- 下载中状态栏显示进度条 + 已下载/总大小 + 已用时（`M:SS` 格式，从下载开始计时）：`下载中 file.txt [=====-----] 45% 1.2M/2.7M 用时0:12`（无 `Content-Length` 时只显示已下载字节数 + 用时，不显示百分比/总大小）
- 进度回调按时间节流重绘（避免高频重绘导致闪烁）
- 下载成功后状态栏提示 `已下载到 <本地路径>（用时 M:SS）`，返回浏览界面，光标停留在原条目
- 状态栏的临时提示（加载失败、"已在根目录"、下载取消/失败/成功等）以居中高亮的形式短暂显示，一段时间后自动消失，恢复为常驻的按键提示行；不影响下方已加载的目录列表

## CLI 参数

```
python3 nginx_tui.py <URL> [--output-dir DIR] [--insecure]
```
- `URL`：必填；缺少协议前缀（`http://`/`https://`）时自动补 `http://`
- `--output-dir`/`-o`：可选，默认当前工作目录
- `--insecure`/`-k`：可选，跳过 HTTPS 证书校验（用于自签名证书的服务器），默认不开启；一旦开启，加载/刷新/下载都不校验证书，有中间人攻击风险，仅应在信任目标网络时使用
- 不带任何参数运行（`python3 nginx_tui.py`）：直接打印完整帮助信息（等价于 `--help` 的输出：用法、参数说明、示例）并退出，不是 argparse 默认的简短报错
- `-h`/`--help`：同上，随时可查看帮助

## 错误处理

- 目录加载失败（网络错误、HTTP 非 200）：状态栏报错，停留在当前目录，不写入历史栈；页面解析不到任何链接则当作空目录处理，不算失败（见架构与组件一节）
- 加载/刷新目录列表、下载文件时，状态栏提示文字都带「（Ctrl-C 取消）」的提示；加载/刷新提示里显示的是目录/文件名（进入子目录时取该条目的名字，刷新/启动时取 URL 最后一段路径），而不是完整 URL——嵌套很深的路径本身可能很长，用短名字避免提示文字本身就把关键信息挤出屏幕
- 加载目录列表（进入子目录）等待期间按 Ctrl-C：视为取消，不写入历史栈，直接停留/返回到发起加载前所在的上一级目录；应用启动时的首次加载被取消则打印取消信息并以退出码 130 退出（与其他 Ctrl-C 退出路径一致），区别于加载真的失败（网络错误等）时的退出码 1
- 刷新当前目录列表等待期间按 Ctrl-C：视为取消，保留刷新前已缓存的旧列表，停留在当前目录
- 空闲浏览（没有加载/刷新/下载/覆盖确认在等待中）时按 Ctrl-C：不退出程序，直接忽略；退出程序只能用 `q`/`Q`；这个保护覆盖主循环一整轮按键处理（画面重绘、读键、动作分发全部在同一个 try 里），不只是读键那一步，移动/翻页/返回上级这些没有自己 Ctrl-C 处理逻辑的动作也不会被 Ctrl-C 打断退出
- 下载失败/中断（Ctrl-C、断网、磁盘写入失败/权限不足）：清理 `.part` 残留文件，状态栏报错，回到可操作的浏览界面
- 加载/刷新/下载这类会阻塞等待网络的操作结束后，会丢弃等待期间积压的按键/鼠标输入，不会在操作完成的瞬间把用户在等待时随手按的键/点的坐标"补放"出来当成新的操作
- 覆盖确认提示（y/n）等待期间按 Ctrl-C：视为取消，回到浏览界面，不退出程序；等待期间收到终端 resize 或鼠标事件不算作"按了键"，重绘（resize 时）或忽略（鼠标）后继续等待真正的按键回答
- 下载的临时文件只在本次下载确实创建/打开过 `.part` 文件后才会在失败时清理，不会误删本机已存在、并非本次下载产生的同名 `.part` 文件
- 启动参数错误（`--output-dir` 无法创建，如权限不足）：进入 TUI 前打印错误信息到 stderr，以非零退出码退出；`--output-dir` 支持 `~` 展开为用户主目录
- 系统 locale 不受支持（`locale.setlocale` 失败）：不阻断启动，忽略后继续（极端情况下宽字符可能显示异常）
- 终端不支持光标可见性控制（`curs_set` 失败）等次要 curses 初始化调用失败：忽略，不影响正常使用
- 终端 resize：捕获 `KEY_RESIZE`，重新计算布局并重绘；单次绘制中任何终端边界写入异常只跳过该帧重绘，不影响程序继续运行；resize 或刷新导致列表变短时，滚动位置会同时向下裁剪，不会停留在已经超出列表末尾的位置留下空白
- 终端窗口太小（高度 < 4 行，或宽度小于名称/大小/修改时间三列能不换行容纳的最小宽度）：不再尝试挤出一个会换行/被截断的列表，直接在屏幕正中央显示「窗口太小，按 q 退出」；这个分支不会绘制底部按键提示行，所以退出方式必须写进提示文字本身——否则用户在这个状态下无从得知还能做什么；`q` 在这个状态下依然正常生效（按键处理和绘制是分开的，不受当前画的是什么影响），只是不可见；提示文字本身刻意写得很短——它是在"空间已经不够"的场景下显示的，如果文字太长在临界宽度下自己也会被截断到看不出意思，所以要能在比正常状态窄得多的宽度下依然完整显示

## 测试

- 使用用户本地 docker nginx（`autoindex on;`）手动验证：多级目录嵌套、含空格/中文/特殊字符的文件名、空目录、大文件下载进度显示、下载中 Ctrl-C 中断后 `.part` 文件被清理、从子目录一路返回到根目录后再按返回（应无操作且不报错）、终端 resize
- `parse_index` 是纯函数，不依赖 curses，后续如需要可以单独用静态 HTML fixture 做单元测试；本次不预先编写自动化测试
